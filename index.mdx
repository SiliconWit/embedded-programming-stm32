Step up from 8-bit AVR to 32-bit ARM Cortex-M. The STM32 family powers everything from consumer electronics to industrial controllers and medical devices. This course teaches you to program the STM32F103 (Blue Pill) using both HAL and direct register access, so you always understand what the abstraction hides. Each lesson builds a real project. #STM32 #ARM #EmbeddedSystems

## Why STM32?

<CardGrid>
  <Card title="Industry Standard Architecture" icon="approve-check">

  <Card title="HAL and Registers, Side by Side" icon="open-book">

## Course Structure

Each lesson follows a consistent cycle:

<Steps>
1. **The Project**
   A real, useful thing to build. The project drives which peripheral concepts you need.

2. **Architecture and Registers**
   How this peripheral works in the STM32 reference manual. Clock enables, register maps, configuration bits.

3. **HAL Implementation**
   Build it first with STM32 HAL for a working baseline. Understand the CubeMX configuration.

4. **Register-Level Insight**
   See exactly what the HAL functions do under the hood. Rewrite critical sections at the register level.

5. **Build and Debug**
   Flash via SWD, test on real hardware, measure with logic analyzer or oscilloscope when relevant.
</Steps>

## Lessons

<CardGrid>
  <Card title="Lesson 1: STM32 Toolchain and ARM Architecture" icon="star">
    [STM32 Toolchain and ARM Architecture](./stm32-toolchain-arm-architecture). Set up arm-gcc, OpenOCD, and STM32CubeMX. Understand Cortex-M3 architecture: vector table, startup code, linker scripts, CMSIS. Build a breathing LED that demonstrates precise clock configuration.
    **Build:** Breathing LED with HSE clock. **Parts:** Blue Pill, ST-Link V2, breadboard, LED.
  </Card>

  <Card title="Lesson 2: GPIO and Clock Tree Configuration" icon="document">
    [GPIO and Clock Tree Configuration](./gpio-clock-tree-configuration). Master the RCC clock tree, AHB/APB buses, GPIO modes, and alternate functions. Build a rotary encoder menu system controlled through a serial terminal.
    **Build:** Rotary encoder menu system. **Parts:** KY-040 rotary encoder.
  </Card>

  <Card title="Lesson 3: Timers, PWM, and Input Capture" icon="document">
    [Timers, PWM, and Input Capture](./timers-pwm-input-capture). Explore general-purpose and advanced timers: PWM generation, output compare, input capture, encoder interface. Build a servo-driven pan mechanism controlled by the rotary encoder.
    **Build:** Servo pan mechanism. **Parts:** SG90 micro servo.
  </Card>

  <Card title="Lesson 4: UART with DMA and Interrupts" icon="document">
    [UART with DMA and Interrupts](./uart-dma-interrupts). Configure USART with NVIC interrupt priorities and DMA channels. Implement circular buffers, idle line detection, and a command parser. Build an interactive command shell.
    **Build:** Interactive serial command shell. **Parts:** Reuse existing.
  </Card>

  <Card title="Lesson 5: SPI and I2C: HAL vs Register Level" icon="document">
    [SPI and I2C: HAL vs Register Access](./spi-i2c-hal-register-access). Implement both protocols using HAL, then rewrite the drivers at register level. Compare code size, speed, and clarity. Build a sensor dashboard on OLED.
    **Build:** Sensor dashboard on OLED. **Parts:** SSD1306 OLED, BME280.
  </Card>

  <Card title="Lesson 6: ADC with DMA and Analog Watchdog" icon="document">
    [ADC with DMA and Analog Watchdog](./adc-dma-analog-watchdog). Configure multi-channel ADC scanning with DMA transfers and analog watchdog thresholds. Build a voltage and current monitor with visual alerts.
    **Build:** Voltage/current monitor. **Parts:** INA219 current sensor module, potentiometer.
  </Card>

  <Card title="Lesson 7: Debugging with SWD and Fault Handlers" icon="document">
    [Debugging with SWD, GDB, and Fault Handlers](./debugging-swd-gdb-fault-handlers). Master SWD debugging: breakpoints, watchpoints, register inspection, HardFault decoding, and ITM trace output. Receive intentionally buggy firmware and diagnose every fault.
    **Build:** Bug hunt challenge (5 bugs to find and fix). **Parts:** No new parts.
  </Card>

  <Card title="Lesson 8: FreeRTOS on STM32" icon="document">
    [FreeRTOS Fundamentals on STM32](./freertos-fundamentals-stm32). Introduce real-time multitasking: tasks, queues, semaphores, mutexes, and priority management. Build a multitasking dashboard with sensor reading, display, and serial logging as separate tasks.
    **Build:** Multitasking sensor dashboard. **Parts:** Reuse existing.
  </Card>

  <Card title="Lesson 9: Low-Power and Production Firmware" icon="document">
    [Low-Power Modes and Production Firmware](./low-power-production-firmware). Explore Stop/Standby modes, RTC wakeup, flash and option bytes, and bootloader basics. Build a battery-powered data logger that wakes periodically, logs to SPI flash, and sleeps.
    **Build:** Battery-powered data logger. **Parts:** CR2032 holder, W25Q SPI flash module.
  </Card>

## Getting Started

<Steps>
1. **Get a Blue Pill board** (STM32F103C8T6) and an **ST-Link V2 clone**. Together they cost under $8.

2. **Install the toolchain**: arm-none-eabi-gcc, OpenOCD, and STM32CubeMX. Lesson 1 covers setup on Linux, macOS, and Windows.

3. **Download the STM32F103 reference manual** (RM0008) from [ST Microelectronics (PDF)](https://www.st.com/resource/en/reference_manual/cd00171190-stm32f101xx-stm32f102xx-stm32f103xx-stm32f105xx-and-stm32f107xx-advanced-arm-based-32-bit-mcus-stmicroelectronics.pdf) ([product page](https://www.st.com/en/microcontrollers-microprocessors/stm32f103c8.html)). This is your primary reference, covering every register in the chip.

4. **Start with Lesson 1.** Wire the ST-Link to the Blue Pill and flash your first program.

5. **Work through lessons in order.** Each builds on peripherals and patterns from previous lessons.
</Steps>

:::note[Blue Pill or Black Pill?]
This course uses the Blue Pill (STM32F103C8T6, Cortex-M3, 72 MHz, 64 KB flash). The Black Pill (STM32F411CEU6, Cortex-M4F, 100 MHz, 512 KB flash) is more powerful with an FPU and more memory, and every lesson in this course works on it with minor pin and clock adjustments. If you plan to continue beyond this course, the Black Pill gives you more room to grow. Either board is a fine starting point.
:::

:::tip[Coming from Arduino?]
If you completed the [Embedded Programming: ATmega328P](/education/embedded-programming-atmega328p/) course, the jump to STM32 will feel natural. The concepts are the same (GPIO, timers, UART, SPI, I2C), but the STM32 adds clock trees, DMA, and a more sophisticated interrupt controller. This course explicitly bridges the two worlds.
:::
